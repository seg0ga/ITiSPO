# Вопросы для самопроверки

1. **Что такое слой в Docker и почему порядок команд в Dockerfile важен для кэширования?**

    Слой - это результат выполнения одной инструкции. Docker кэширует слои и использует их при пересборке, а порядок важен, тк изменение команды ломает кэш для нее и всех последующих слоев
2. **В чем суть Multi-stage build? Какие проблемы он решает?**

    Несколько этапов сборки в одном Dockerfile. На первом этапе с полными инструментами собирается приложение, на финальный этап копируется только готовый рез-т, что уменьшает размер образа и убирает из него компиляторы и временные файлы
3. **Зачем нужен файл `.dockerignore`?**

    Он нужен, чтобы исключить ненужные файлы из контекста сборки, это ускоряет сборку и предотвращает паопадание в образ логов, локальных зависимостей и тд.
4. **Чем отличаются инструкции `CMD` и `ENTRYPOINT`?**

    Entrypoint задает исполняемую команду, которую сложно переопределить, а cmd задает аргументы по умолчанию, которые можно легко заменить при запуске
5. **Почему рекомендуется использовать `COPY` вместо `ADD`?**
    
    copy делает только копирование файлов, а Add дополнительно умеет распаковывать архивы и скачивать по URL, что может привести к неожиданному поведению
6. **Почему приложение в контейнере должно слушать `0.0.0.0`, а не `127.0.0.1`?**

   127.0.0.1 слушает только внутри контейнера, а 0.0.0.0 слушает все сетевые интерфейсы, поэтому подключаться к контейнеру можно как с хостовой машини, так и с других контейнеров
