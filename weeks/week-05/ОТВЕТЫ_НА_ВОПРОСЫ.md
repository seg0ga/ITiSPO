# Вопросы для самопроверки

1. **В чем главное концептуальное отличие GraphQL от REST?**

    Главное концептуальное отличие GraphQL от REST заключается в том, что у REST много эндпоинтов, и каждый возвращает только один готовый набор данных, а в GraphQL всего один адрес, и он возвращает те данные, которые нам нужны.
2. **Что такое Schema Definition Language (SDL)?**

    Это язык описания данных. На нем описывается то, какие данные умеет отдавать сервси
3. **Чем Query отличается от Mutation? Могут ли они выполняться параллельно?**

    Query нужен для получения данных, а Mutation для их изменения, добавления и удаления. Query парралельно выполняться может, для скорости, так как мы только считываем данные, а Mutation не может, все идет по очереди, чтобы не перепутать изменения, добавления или удаления
4. **Что такое "резолвер" (resolver) и кто его вызывает?**

    Это функция, которая нужна для получения данных для полей в GraphQL, который его и вызывает
5. **Как решается проблема N+1 запросов в GraphQL (dataloader)?**

    Проблема N+1 запросов в GraphQL решается благодаря dataloader. Он группирует запросы, из-за чего загружает данные за один SQL запрос. Это снижает нагрузки на сервер и БД
6. **Почему в GraphQL сложно реализовать кэширование на уровне HTTP (как в REST)?**

    В GraphQL сложно реализовать кэширование на уровне HTTP как в REST, тк в REST у каждой записи свой адрес, те можно просто сохранить ответ с каждого адреса, а в GraphQl все запросы приходят на один адрес, но с разными телами запросов
