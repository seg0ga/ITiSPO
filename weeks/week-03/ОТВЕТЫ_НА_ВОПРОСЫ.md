# Вопросы для самопроверки

1. В чем разница между Forward Proxy (обычный прокси) и Reverse Proxy (обратный прокси)?

    Forward Proxy работает на стороне клиента и скрывает его от сервера. Reverse Proxy работает на стороне сервера и скрывает серверы от клиента.
2. Что такое Upstream в терминологии Nginx? 

    Это блок в конфиге Nginx, где перечислены серверы приложений. Когда приходит запрос, Nginx выбирает одного из этого списка и пересылает ему запрос. По сути, просто список адресов бэкендов...
3. Зачем скрывать реальные IP-адреса сервисов от клиента?

    Чтобы клиенты не имели прямого доступа к бэкендам в обход шлюза и систем безопасности.Если реальные IP серверов приложений не публикуются наружу, атаковать их напрямую невозможно...
4. Какие еще задачи, кроме маршрутизации, может выполнять API Gateway (назовите хотя бы 2)?

    Аутентификация и авторизация запросов, сбор аналитики
5. Что произойдет, если Gateway упадет? Как можно от этого защититься?

    Gateway становится единой точкой отказа — если он умрет, сервис будет недоступен. Защита — несколько Gateway'ев и балансировщик между ними, который направляет трафик только на живые Gateway'и
