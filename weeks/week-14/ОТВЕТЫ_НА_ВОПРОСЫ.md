# Вопросы для самопроверки

1. **В чем разница между Continuous Integration, Continuous Delivery и Continuous Deployment?**

   - Continuous Integration - это процесс автоматической сборки и тестирования кода при каждом изменении в репозитории
   - Continuous Delivery - это следующий этап
   - Continuous Deployment - это полная автоматизация, при которой каждое успешное изменение автоматически попадает в продакшен без участия человека
2. **Почему тесты должны запускаться *до* сборки Docker образа (или хотя бы до его пуша)?**

    Чтобы не тратить время на сборку и отправку образа, если тесты все равно упадут, и чтобы не засорять registry непроверенными артефактами
3. **Зачем нужно кэширование зависимостей в CI пайплайне?**

    Чтобы не скачивть одни и те же библиотеки при каждом запуске, сокращая время выполнения пайплайна
4. **Что такое "Secret" в контексте CI/CD? Как безопасно передать пароль от Docker Hub в пайплайн?**

    Secret - это конфиденциальные данные. Пароль передают черех встроенное хранилище секретов CI-системы, создавая зашифрованную переменную, а в пайплайне подставлятю ее через docker login
5. **Что такое "Matrix Build" (матричная сборка)?**

    это паралельный запуск одной задачи с разными параметрами, например, тестирование на нескольких версиях питона и ОС одновременно без дублирование кода пайплайна
6. **Почему нельзя хранить конфиги продуктовой среды в репозитории кода (или можно, но осторожно)?**

    Потому что при утечки репозитория зллоумышленник получит доступ ко всей инфраструктуре, конфиши можно хранить только в зашифрованном виде или во внешних хранилищах
