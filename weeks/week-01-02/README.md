# Знакомство с Docker


Критерии оценки практической работы
|Уровень|Требования к выполнению|Прогресс|
|:--:|--|:--:|
|Базовый|Сервис написан на FastAPI, Dockerfile успешно собирается, CRUD-операции работают в Swagger UI.|🟢|
|Хорошо|Оптимизирован Dockerfile (разделение установки зависимостей и кода), настроен .dockerignore, используются корректные HTTP-коды ответов.|🟢|
|Отлично|Реализована обработка исключений, сервис запускается под non-root пользователем, представлен файл docker-compose.yml для запуска сервиса.|🟢|
