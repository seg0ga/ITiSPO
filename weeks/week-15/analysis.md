# Latency vs Throughput Analysis

### tasks-s05

## Условия теста
- **Tool**: Python ThreadPoolExecutor (wrk/ghz стиль)
- **Duration**: 30s
- **Connections**: 10, 100, 1000
- **REST**: FastAPI (week-01-02), порт 8208
- **gRPC**: gRPC Python (week-07), порт 50051

## Выводы
1. Точка насыщения для gRPC находится примерно на 1800 RPS
2. REST показал более высокий RPS при 1000 соединениях, но с нестабильным latency
3. При увеличении нагрузки Latency растет экспоненциально
4. gRPC стабильнее при высоких нагрузках, тогда как REST показывает большой разброс на P95 и P99
5. Для высоконагруженных систем рекомендуется gRPC благодаря стабильному latency и меньшему потреблению ресурсов
