# Вопросы для самопроверки

1. **Какие критерии вы использовали при разделении монолита на микросервисы (или при проектировании сервисов)?**

    Я разделил систему по ролям: gateway принимает внешние запросы, logs-svc-s05 работает с логами, log-notifier получает внутренние gRPC-уведомления
2. **Почему вы выбрали именно этот стек технологий (язык, БД, протоколы)?**

    Я выбрал Python и FastAPI, потому что этот стек уже использовался в прошлых лабах....
3. **Как ваши сервисы справляются с ошибками (retry, circuit breaker, graceful degradation)?**

    Практически никак: если лог не найден - возвращается 404, если gRPC-сервис недоступен - основной сервис всё равно продолжает работать
4. **Как организован деплой и обновление вашей системы без простоя (Zero Downtime)?**

    Запуск и деплой сделаны через docker compose, Zero Downtime нет
5. **Что было самым сложным в интеграции всех компонентов вместе?**

    Самым сложным было связать REST, gRPC и Docker Compose в одном проекте.....,
6. **Что бы вы улучшили, если бы у вас был еще месяц?**

    БД сделал бы :)
