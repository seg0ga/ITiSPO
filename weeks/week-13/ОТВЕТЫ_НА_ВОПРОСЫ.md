# Вопросы для самопроверки

1. **Зачем нужен Helm? Какую проблему он решает?**

    Helm - это менеджер пакетов для Kubernates. Он решает проблему управления множеством Kubernetes-манифестов через шаблонизацию, а так же контроль версий с возможностью отката
2. **Как работает приоритет значений (Values)? Кто победит: `values.yaml` или флаг `--set`?**

    вначале применяются значения из values.yaml, затем из файлов, переданных через -f, а поверх них флаги --set, которые имеют наивысший приоритет
3. **Что такое функция `include` и `_helpers.tpl`? Зачем они нужны?**

    - include - это функция для вызова _helpers.tpl
    - _helpers.tpl - это файл для определения переиспользуемых шаблонов
4. **Почему в шаблонах Helm так важно следить за отступами и зачем нужна функция `indent` или `nindent`?**

    yaml требует строгого соблюдения пробелов. Для этого используется nintent , который вставляет перевод строки и нужное количество пробелов, сохраняя валидность манифеста
5. **Как откатить неудачный релиз с помощью Helm?**

    `helm rollback _имя-релиза_ _номер-версии_`
6. **Можно ли использовать Helm только для генерации YAML, без установки в кластер?**

    Да, можно. Для этого существует команда helm template, которая выполняет рендеринг шаблонов
