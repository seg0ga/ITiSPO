# Вопросы для самопроверки

1. **Зачем создавать пользовательскую сеть (bridge network) в Docker Compose, если есть дефолтная?**

    Пользовательская сеть дает возможность сервисам общаться друг с другом по именам. В дефолтной сети это работает не так надежно и предсказуемо
2. **В чем разница между `build` и `image` в описании сервиса?**

    build - это указание Docker собрать образ из Dockerfile (путь к контексту сборки), а image - использование готовго образа из реестра
3. **Что делает `depends_on`? Гарантирует ли он, что приложение в зависимом контейнере полностью инициализировалось?**

    depends_on гарантирует только то, что зависимый контейнер создан и запущен (но не то что его приложение готово принимать запросы), полной инициализации он не гарантирует
4. **Как работают healthcheck-и и как использовать их в `depends_on`?**

    healthcheck - это периодическая проверка, если она несколько раз подряд проваливается, то контейнер помечается как unhealthy. в depends_on можно добавить condition, тогда сервис запустится только после того, как контейнер станет healthy
5. **Как передать переменные окружения в контейнер через Compose (минимум 2 способа)?**

    - через environment (прямо в yaml)
    - через env_file (указать путь к файлу .env)
6. **Почему Docker Compose редко используют в продакшене (хотя это возможно)?**

    Для продакшена нужны нормальная оркестрация и управление кластером (баланировка нагрузки, перезапуск при падении, масштабирование), а DockerCompose это не позволяет. Он удобен для локальной разработки
