# Вопросы для самопроверки

1. **Что такое Server Streaming? Приведите пример использования.**

    Server Streaming - это когда клиент делает один запрос, а сервер отправляет поток сообщений, например подписка на ленту новостей
2. **Чем Server Streaming отличается от просто получения большого списка (List) в одном ответе?**

    Главное отличие заключается в том, то при стрименге клиент получает данные по мере их появления, а не дожидается полного ответа
3. **Какие метрики производительности существуют (Latency, Throughput, CPU Usage)?**

    Основные метрики производительности - latency (время ожидания ответа), throuhnput(пропускная способность или количество запросов в секунду) и CPU usage (нагрузка на процессор)
4. **Почему важно переиспользовать канал (Channel) в gRPC клиенте при замерах производительности?**

    При замерах важно использовать один канал, тк создание нового требует установки соединения, что немного искажает реальные данные
5. **В каких сценариях REST может оказаться быстрее или удобнее gRPC?**

    REST лучше всего использовать для простых API, например при работе через браузер или когда важна читаемость JSON
6. **Что такое "Head-of-Line Blocking" и как HTTP/2 (и gRPC) решает эту проблему?**

    Head-of-Line Blocking - это когда медленный запрос блокирует остальные. HTTP2 и Grpc решают это через мультиплексирование, позволяя передавать множество потоков одновременно
